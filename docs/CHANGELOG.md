# 変更履歴

## Phase 6

- GTFS `parent_station` による親停留所単位のグループ化を追加
- 子停留所名が数字のみの場合に「○番のりば」へ正規化
- 停留所カード表面へ代表行き先を追加
- 複数のりばを閉じた状態から展開する表示へ変更
- のりば一覧をツリー風の階層表示へ変更
- お気に入り操作後も停留所カードの開閉状態を保持
- データschema versionを5へ更新
- Service Workerキャッシュを `tobus-navi-v7` に更新

## Phase 5

- 本日の全時刻表を `<details>` による開閉形式へ変更
- 時刻表は開いた時にだけ生成する遅延描画へ変更
- 次の発車予定は常時表示を維持
- GTFS-RT通信へ10秒タイムアウトを追加
- 複数取得元の順次フォールバックを追加
- リアルタイム通信の重複実行を防止
- 失敗時の指数的バックオフを追加
- オンライン復帰時の即時更新を追加
- バックグラウンド中の自動更新停止と復帰時更新を追加
- 古いフィードと古い車両情報の判定を追加
- 取得元、最終更新時刻、再試行時刻の表示を追加
- Cloudflare Workerへ上流タイムアウトと障害時キャッシュを追加
- Ubuntu用リアルタイムHTTP/CORS診断スクリプトを追加
- Service Workerキャッシュを `tobus-navi-v6` に更新
